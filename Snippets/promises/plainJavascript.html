Promises at Low Level

Promise - The Promise object represents the eventual completion (or failure) of an asynchronous operation and its resulting value.

Promise is having three states

  1. Pending
  2. Fulfilled
  3. Rejected

If someone is saying promise is settled or resolved it means promise is either in fullfilled state or rejected state but no longer in pending state.

Javascript promises are eagerly evaluated, promises are not lazy

Promises are Objects

Promises consumes only one function as a argument called executor function
